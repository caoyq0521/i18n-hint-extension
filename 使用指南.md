# i18n 翻译提示插件 - 使用指南

## 快速开始

### 1. 安装依赖和编译

在终端中执行：

```bash
cd i18n-hint-extension
chmod +x install.sh
./install.sh
```

或者手动执行：

```bash
cd i18n-hint-extension
npm install
npm run compile
```

### 2. 启动调试

1. 在 VSCode 中打开 `i18n-hint-extension` 目录
2. 按 `F5` 或点击"运行" -> "启动调试"
3. 会打开一个新的 VSCode 窗口（扩展开发主机）
4. 在新窗口中打开你的项目

### 3. 测试插件

在你的代码中找到使用 `t()` 函数的地方，例如：

```typescript
title: t('business.statistics.column.projectType'),
```

你会看到：

**行内提示效果：**
```typescript
title: t('business.statistics.column.projectType'), // 项目类型
```

**悬停提示效果：**
鼠标悬停在 `t('business.statistics.column.projectType')` 上时，会显示：
```
i18n 翻译

Key: business.statistics.column.projectType

中文: 项目类型
```

## 功能说明

### 1. 行内翻译提示

- **触发条件**: 代码中包含 `t('key')` 或 `t("key")`
- **显示位置**: 在函数调用后面
- **显示格式**: `// 中文翻译`
- **配置项**: `i18nHint.enableInlineHints`

### 2. 悬停翻译提示

- **触发条件**: 鼠标悬停在 `t('key')` 上
- **显示内容**: 
  - 翻译 key
  - 中文翻译
- **配置项**: `i18nHint.enableHover`

### 3. 自动更新

当你修改翻译文件时，插件会自动重新加载翻译内容，无需重启 VSCode。

## 配置说明

在 VSCode 设置中搜索 `i18nHint` 可以找到以下配置项：

### i18nHint.localesPath
- **默认值**: `"src/locales/lang"`
- **说明**: 翻译文件所在的相对路径（相对于工作区根目录）
- **示例**: 如果你的翻译文件在 `src/i18n/locales`，则设置为 `"src/i18n/locales"`

### i18nHint.defaultLocale
- **默认值**: `"zh-CN"`
- **说明**: 默认显示的语言
- **可选值**: `"zh-CN"`, `"en"` 等（取决于你的翻译文件目录名）

### i18nHint.enableInlineHints
- **默认值**: `true`
- **说明**: 是否启用行内提示
- **建议**: 如果觉得行内提示影响代码阅读，可以关闭

### i18nHint.enableHover
- **默认值**: `true`
- **说明**: 是否启用悬停提示
- **建议**: 建议保持开启，不影响代码阅读

## 支持的文件类型

- TypeScript (`.ts`)
- Vue (`.vue`)
- JavaScript (`.js`)
- TypeScript React (`.tsx`)
- JavaScript React (`.jsx`)

## 翻译文件要求

### 文件结构

```
src/locales/lang/
├── zh-CN/
│   ├── business/
│   │   ├── statistics.ts
│   │   ├── project.ts
│   │   └── ...
│   ├── common.ts
│   └── ...
└── en/
    └── ...
```

### 文件格式

翻译文件必须是 TypeScript 格式，使用 `export default` 导出对象：

```typescript
export default {
  column: {
    projectType: '项目类型',
    projectName: '项目名称',
    // ...
  },
  // ...
};
```

### 翻译 Key 格式

使用点分隔符表示层级关系：

```typescript
// 文件: src/locales/lang/zh-CN/business/statistics.ts
export default {
  column: {
    projectType: '项目类型',
  }
};

// 使用: t('business.statistics.column.projectType')
// 显示: 项目类型
```

## 常见问题

### Q: 翻译不显示怎么办？

**A**: 请检查以下几点：

1. **翻译文件路径是否正确**
   - 检查 `i18nHint.localesPath` 配置
   - 确认翻译文件确实存在于该路径下

2. **翻译 key 是否正确**
   - 检查 key 的拼写
   - 确认 key 在翻译文件中存在

3. **翻译文件格式是否正确**
   - 必须使用 `export default { ... }` 格式
   - 确保没有语法错误

4. **查看控制台错误**
   - 打开 VSCode 开发者工具：`帮助` -> `切换开发人员工具`
   - 查看 Console 标签页是否有错误信息

### Q: 性能问题怎么办？

**A**: 如果项目很大，翻译文件很多，可以：

1. 禁用行内提示，只使用悬停提示
2. 优化翻译文件的组织结构
3. 减少翻译文件的嵌套层级

### Q: 如何调试插件？

**A**: 

1. 在插件代码中设置断点
2. 按 `F5` 启动调试
3. 在扩展开发主机窗口中触发功能
4. 回到插件开发窗口查看断点

### Q: 如何打包分发？

**A**:

```bash
# 安装打包工具
npm install -g @vscode/vsce

# 打包
cd .vscode-extension
vsce package

# 会生成 i18n-hint-extension-1.0.0.vsix 文件
```

然后可以通过以下方式安装：
- VSCode 命令面板 -> "从 VSIX 安装扩展"
- 或者分享给团队成员安装

## 技术原理

### 1. 翻译文件加载

- 扫描 `localesPath` 目录下的所有 `.ts` 文件
- 解析文件内容，提取翻译对象
- 按照文件路径构建翻译 key 的层级结构

### 2. 文件监听

使用 VSCode 的 `FileSystemWatcher` API 监听翻译文件的变化：
- 文件修改时重新加载
- 文件创建时重新加载
- 文件删除时重新加载

### 3. 行内提示

使用 VSCode 的 `InlayHintsProvider` API：
- 扫描文档中的 `t('key')` 模式
- 查找对应的翻译
- 在合适的位置插入提示

### 4. 悬停提示

使用 VSCode 的 `HoverProvider` API：
- 检测鼠标位置是否在 `t('key')` 上
- 查找对应的翻译
- 显示 Markdown 格式的提示信息

## 进阶使用

### 自定义配置文件

在工作区的 `.vscode/settings.json` 中添加：

```json
{
  "i18nHint.localesPath": "src/locales/lang",
  "i18nHint.defaultLocale": "zh-CN",
  "i18nHint.enableInlineHints": true,
  "i18nHint.enableHover": true
}
```

### 团队共享配置

将配置添加到项目的 `.vscode/settings.json`，提交到版本控制系统，团队成员就能共享相同的配置。
